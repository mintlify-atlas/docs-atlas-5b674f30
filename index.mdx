title: "Distroless: language-focused container base images"
description: "Distroless images contain only your application and its runtime dependencies — no shells, no package managers, minimal attack surface."
---

Distroless images are minimal container base images built by Google that include only your application and its runtime dependencies. By stripping out shells, package managers, and all other OS-level utilities, distroless images significantly reduce your attack surface, improve CVE signal-to-noise, and shrink image sizes dramatically.

## Get started

<Steps>
  <Step title="Pick the right base image">
    Choose from `static`, `base`, `cc`, or a language-specific image based on your application's runtime needs. See [available images](/images/overview) for a full list.
  </Step>
  <Step title="Write a multi-stage Dockerfile">
    Use a full build image to compile your app, then copy only the binary into a distroless image for the final stage.

    ```dockerfile
    FROM golang:1.22 as build
    WORKDIR /go/src/app
    COPY . .
    RUN go mod download
    RUN CGO_ENABLED=0 go build -o /go/bin/app

    FROM gcr.io/distroless/static-debian12
    COPY --from=build /go/bin/app /
    CMD ["/app"]
    ```
  </Step>
  <Step title="Verify the image signature">
    All distroless images are signed with cosign. Verify before use:

    ```bash
    cosign verify gcr.io/distroless/static-debian12 \
      --certificate-oidc-issuer https://accounts.google.com \
      --certificate-identity keyless@distroless.iam.gserviceaccount.com
    ```
  </Step>
  <Step title="Build and run">
    ```bash
    docker build -t myapp .
    docker run myapp
    ```
  </Step>
</Steps>

<Note>
  Distroless images do not contain a shell by default. Always use the **vector form** for `CMD` and `ENTRYPOINT` instructions in your Dockerfile (e.g., `CMD ["/app"]` not `CMD /app`).
</Note>
